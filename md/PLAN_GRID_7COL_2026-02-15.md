# 계획: 카드 그리드 6열 → 7열 변경

> 작성일: 2026-02-15

---

## 현재 상태 분석

| 항목 | 값 |
|------|-----|
| 컨테이너 max-width | 1400px |
| 컨테이너 padding | 좌우 20px (p-5) |
| 실제 그리드 영역 | 1360px |
| 카드 너비 (--card-width) | 200px |
| 카드 높이 (--card-height) | 320px |
| 가로 간격 (gap-x) | 25px |
| 그리드 방식 | auto-fill, minmax(200px, 1fr) |

**현재 6열 계산:** 200px x 6 + 25px x 5 = 1325px (1360px 안에 들어감)
**7열 시도:** 200px x 7 + 25px x 6 = 1550px (1360px 초과, 안 들어감)

---

## 해결 방안 비교

### A안: 컨테이너 확장 (max-w 1600px)

- 컨테이너를 1600px로 넓힘
- 실제 영역: 1600 - 40 = 1560px
- 7열: 200 x 7 + 25 x 6 = 1550px (들어감)
- 카드 크기 유지, 전체 레이아웃만 넓어짐
- 영향: Header, Footer, 검색바, ResultBar 등 모든 max-w-[1400px] 변경 필요

### B안: 카드 크기 축소 (--card-width 170px)

- 카드 너비를 170px로 축소
- 7열: 170 x 7 + 25 x 6 = 1340px (1360px 안에 들어감)
- 카드 높이도 비율 유지하여 272px로 축소
- 컨테이너 변경 불필요

### C안: 복합 (컨테이너 약간 확장 + 간격 축소)

- max-w를 1500px으로, gap-x를 20px으로
- 실제 영역: 1460px
- 7열: 200 x 7 + 20 x 6 = 1520px (초과)
- 카드 약간 축소 필요 (190px): 190 x 7 + 20 x 6 = 1450px (들어감)

---

## 수정 대상 파일

| 파일 | 수정 내용 |
|------|-----------|
| globals.css | --card-width, --card-height 값 변경 (B안 선택 시) |
| 또는 여러 컴포넌트 | max-w-[1400px] → max-w-[1600px] 변경 (A안 선택 시) |

**모바일은 변경 없음** — 768px 이하는 기존대로 2열 유지.

---

## 영향 범위

- MemberGrid.tsx: auto-fill 방식이므로 카드/컨테이너 크기만 바꾸면 자동 적용
- TeamSection.tsx: 동일한 그리드 클래스 사용, 자동 적용
- MemberCard.tsx: --card-width 참조하므로 자동 적용
- 모바일: 변경 없음
